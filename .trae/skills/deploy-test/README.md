# 部署测试工具

一个用于项目部署前的自动化测试 CLI 工具，支持本地测试、构建验证、线上检查和功能测试四个阶段。

## 功能特性

- **四阶段测试流程**：本地测试 → 构建验证 → 线上检查 → 功能测试
- **进度指示**：每个阶段都有清晰的终端进度提示
- **超时控制**：每个阶段均可配置超时时间，防止长时间挂起
- **格式化报告**：支持带颜色和 emoji 的终端输出，以及 JSON 格式输出
- **诊断建议**：失败时自动提供可能原因和修复步骤
- **进程清理**：自动清理残留子进程，防止资源泄漏
- **零依赖**：仅使用 Node.js 内置模块

## 安装

无需安装，直接克隆或复制到项目目录即可使用。

确保文件结构如下：

```
deploy-test/
├── index.js
├── lib/
│   └── reporter.js
└── README.md
```

## 使用方法

### 基本用法

```bash
node /path/to/deploy-test/index.js
```

### 命令行参数

| 参数 | 简写 | 说明 | 默认值 |
|------|------|------|--------|
| `--url <url>` | - | 指定线上检查的 URL | - |
| `--timeout <ms>` | - | 设置超时时间（毫秒） | 30000 |
| `--skip-build` | - | 跳过构建验证阶段 | false |
| `--skip-online` | - | 跳过线上检查阶段 | false |
| `--verbose` | `-v` | 显示详细错误信息 | false |
| `--json` | - | 以 JSON 格式输出报告 | false |
| `--help` | `-h` | 显示帮助信息 | - |

### 使用示例

**运行全部测试：**

```bash
node index.js --url https://example.com
```

**跳过构建和线上检查（仅运行本地和功能测试）：**

```bash
node index.js --skip-build --skip-online
```

**增加超时时间并输出 JSON 报告：**

```bash
node index.js --url https://example.com --timeout 60000 --json
```

**显示详细错误信息：**

```bash
node index.js --verbose
```

## 测试阶段说明

### 1. 本地测试

自动检测当前目录或上级目录中的 `package.json`，如果存在 `test` 脚本则执行 `npm test`。

- 未找到 `package.json` → 警告跳过
- 未定义 `test` 脚本 → 跳过
- 测试通过 → ✅
- 测试失败 → ❌（附带诊断建议）

### 2. 构建验证

检测 `package.json` 中的 `build` 脚本并执行 `npm run build`。

可通过 `--skip-build` 参数跳过此阶段。

### 3. 线上检查

使用 Node.js 内置 `http` / `https` 模块向指定 URL 发送 GET 请求，检查响应状态码是否在 200-399 范围内。

- 未指定 `--url` → 跳过
- 可通过 `--skip-online` 参数跳过

### 4. 功能测试

检测并执行以下功能测试脚本（按优先级）：

1. `npm run test:e2e`
2. `npm run test:functional`

未找到对应脚本时自动跳过。

## 退出码

| 退出码 | 含义 |
|--------|------|
| 0 | 所有测试通过，可以部署 |
| 1 | 存在失败的测试项，建议修复后再部署 |
| 130 | 用户中断（Ctrl+C） |

## 诊断建议

当某个阶段失败时，工具会自动输出该阶段的可能原因和修复步骤：

```
📋 诊断建议与修复指南

本地测试
  💡 可能原因:
    • 检查本地开发服务器是否已启动
    • 确认端口未被其他进程占用
    • 检查环境变量配置是否正确
  🔧 修复步骤:
    • 运行 npm run dev 启动本地服务器
    • 使用 lsof -i :<port> 查找占用端口的进程
    • 检查 .env 文件中的配置项
```

## 进程清理

工具会监听以下信号和事件，确保子进程被正确清理：

- `process.exit`
- `SIGINT`（Ctrl+C）
- `SIGTERM`

## 技术说明

- **Node.js 版本要求**：Node.js 14 或更高版本
- **无外部依赖**：仅使用 Node.js 内置模块（`http`, `https`, `child_process`, `url`, `path`, `fs`）
- **超时机制**：基于 `Promise.race` 实现，防止测试阶段无限挂起
